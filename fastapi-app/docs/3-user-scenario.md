# 사용자 시나리오 & 사용자 여정 맵
## My TodoList 앱

---

## 1. 주요 사용자 시나리오

### 시나리오 1 — 신규 사용자 회원 가입 및 첫 할 일 등록

**행위자:** 김지은 (대학생, 신규 방문)
**전제 조건:** 계정 없음, 앱 첫 방문

| 단계 | 사용자 행동 | 시스템 반응 |
|------|------------|------------|
| 1 | 앱 URL 접속 | 로그인 페이지로 리다이렉트 |
| 2 | "회원 가입" 링크 클릭 | 회원 가입 폼 표시 |
| 3 | 이메일, 비밀번호 입력 후 가입 | 계정 생성, 로그인 페이지로 이동 |
| 4 | 이메일/비밀번호로 로그인 | JWT 발급, 메인 페이지(할 일 목록) 이동 |
| 5 | "할 일 추가" 클릭 | 추가 폼 표시 |
| 6 | 제목, 완료일, 우선순위 입력 후 저장 | 목록 상단에 새 항목 표시 |

**성공 기준:** 가입 ~ 첫 Todo 등록까지 5분 이내

---

### 시나리오 2 — 기존 사용자 일상적 할 일 관리

**행위자:** 박준호 (고등학생, 기존 사용자)
**전제 조건:** 계정 있음, 기존 Todo 항목 3개 존재

| 단계 | 사용자 행동 | 시스템 반응 |
|------|------------|------------|
| 1 | 앱 접속 후 로그인 | 본인의 Todo 목록 표시 |
| 2 | 기한 초과 항목 확인 | 빨간 강조 항목 시각적으로 인식 |
| 3 | 완료한 항목 체크박스 클릭 | 취소선 + 회색 처리로 완료 표시 |
| 4 | 제목 변경이 필요한 항목 수정 | 수정 폼 표시 → 저장 후 즉시 반영 |
| 5 | 필요 없는 항목 삭제 | 확인 다이얼로그 → 삭제 후 목록 제거 |
| 6 | 새 공부 계획 Todo 추가 | 목록에 추가 |

**성공 기준:** 목록 관리 작업 전체 3분 이내 완료

---

### 시나리오 3 — 기한 초과 항목 처리

**행위자:** 김지은
**전제 조건:** 완료일이 어제인 미완료 항목 존재

| 단계 | 사용자 행동 | 시스템 반응 |
|------|------------|------------|
| 1 | 메인 페이지 접속 | 기한 초과 항목 빨간 강조 표시 |
| 2 | 해당 항목 완료 처리 | 강조 해제, 취소선 처리 |
| 3 | 아직 미완료 항목 완료일 수정 | 수정 저장 후 강조 해제 |

---

### 시나리오 4 — 인증 실패 및 재로그인

**행위자:** 박준호
**전제 조건:** JWT 토큰 만료

| 단계 | 사용자 행동 | 시스템 반응 |
|------|------------|------------|
| 1 | Todo 추가 시도 | 401 오류 반환, 로그인 페이지로 리다이렉트 |
| 2 | 재로그인 | JWT 재발급, 메인 페이지 복귀 |

## 2. 사용자 여정 맵 (User Journey Map)

### 여정 1 — 신규 사용자: 가입 → 첫 Todo 완료

```
단계        | 인지      | 가입      | 로그인    | Todo 추가  | Todo 완료
------------|-----------|-----------|-----------|------------|----------
행동        | 앱 검색   | 가입 폼   | 로그인    | 폼 작성    | 체크박스
            | URL 접속  | 이메일 입력| JWT 발급  | 완료일 설정| 클릭
            |           | 제출      | 메인 이동 | 저장       |
------------|-----------|-----------|-----------|------------|----------
감정        | 😐 궁금   | 🙂 기대   | 😊 기대   | 😊 만족    | 😄 성취감
------------|-----------|-----------|-----------|------------|----------
터치포인트  | 랜딩/     | 가입 폼   | 로그인 폼 | 추가 폼    | 체크박스
            | 로그인 페이지|        |           |            | UI
------------|-----------|-----------|-----------|------------|----------
고충점      | 가입 필요 | 이메일    | 비밀번호  | 완료일     | -
            | 여부 불명 | 중복 오류 | 오류      | 형식       |
------------|-----------|-----------|-----------|------------|----------

            | CTA 버튼  | 오류 표시 |            | 제공       | 피드백
```

## 3. 예외 / 오류 시나리오

| 시나리오 | 트리거 | 시스템 처리 | 사용자 경험 |
|----------|--------|------------|------------|
| 중복 이메일 가입 | 기존 이메일로 가입 시도 | 409 Conflict 반환 | "이미 사용 중인 이메일입니다" 인라인 오류 |
| 잘못된 로그인 정보 | 틀린 비밀번호 입력 | 401 Unauthorized | "이메일 또는 비밀번호가 올바르지 않습니다" |
| 제목 없이 Todo 추가 | 제목 빈 상태로 저장 | 422 Validation Error | "제목은 필수 항목입니다" |
| JWT 만료 | 토큰 유효기간 초과 | 401, 로그인 리다이렉트 | "세션이 만료되었습니다. 다시 로그인해주세요" |
| 존재하지 않는 Todo 수정 | 삭제된 항목 수정 시도 | 404 Not Found | "항목을 찾을 수 없습니다" 토스트 메시지 |
