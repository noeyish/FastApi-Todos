# PRD (Product Requirements Document)
## My TodoList 앱 — v5.0.0

---

## 1. 제품 비전 및 목표

### 비전

"누구나 쉽게 할 일을 관리하고, 기한을 놓치지 않는 개인 생산성 도구"

### 목표

| 목표 | 측정 기준 |
|------|----------|
| 학생 사용자의 과제/일정 관리 효율 향상 | 할 일 완료율 70% 이상 |
| 직관적인 UI로 5분 내 온보딩 완료 | 첫 Todo 등록까지 소요 시간 ≤ 5분 |
| 기한 초과 항목 시각적 구분으로 미처리율 감소 | 기한 초과 인지율 100% |
| 검색으로 빠른 항목 탐색 _(v5)_ | 검색 결과 응답 ≤ 1초 |
| 통계 대시보드로 진행 현황 파악 _(v5)_ | 완료율 즉시 확인 가능 |

## 3. 기능 범위

### ✅ 구현 완료 (v4 이하)

- 회원 가입 (이메일 + 비밀번호)
- 로그인 / 로그아웃 (JWT 인증)
- 할 일 추가 (제목, 설명, 계획완료일, 우선순위)
- 할 일 목록 조회 (본인 것만)
- 할 일 수정 / 삭제
- 완료 상태 토글
- 계획완료일 경과 항목 시각적 강조
- 필터 (전체 / 진행 중 / 완료)
- 우선순위 정렬
- PostgreSQL DB 연동
- Docker + Jenkins CI/CD
- Playwright & Selenium UI 테스트
- SonarQube 정적 분석 환경 구성

### 🆕 v5 신규 기능

- **할 일 검색** ✅: 제목 키워드 실시간 필터링, 검색 버튼, 결과 개수 표시, ✕ 초기화
- **인라인 수정 UI** ✅: 카드 제목 클릭으로 바로 편집, Enter 저장 / ESC 취소
- **통계 대시보드**: 전체/완료/진행 중 개수, 완료율 표시 (미구현)

### ❌ Out of Scope (향후 고려)

- 소셜 로그인 (Google, Kakao)
- 팀/공유 할 일
- 알림/푸시 메시지
- 모바일 네이티브 앱
- 파일 첨부
- 반복 일정 (매주, 매일 등)

| US-06 | 인증된 사용자 | 완료 상태 토글 | 완료한 일을 체크하기 위해 | 즉시 UI 반영 | v3 |

| US-08 | 인증된 사용자 | 할 일 삭제 | 불필요한 항목 제거 | 삭제 확인 후 목록에서 제거 | v3 |

| US-12 | 인증된 사용자 | 통계 대시보드 확인 | 전체 진행 현황을 한눈에 파악 | 완료율, 우선순위별 개수 표시 | v5 (미구현) |
| US-13 | 인증된 사용자 | 인라인 수정 | 수정 화면 전환 없이 빠르게 편집 | 카드에서 바로 제목 수정 가능 | **v5 ✅** |

## 5. 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 응답 속도 | API 응답 ≤ 500ms |
| 보안 | JWT HS256, bcrypt 비밀번호 해시 |
| 테스트 커버리지 | 핵심 API 80% 이상 |
| 정적 분석 | SonarQube Quality Gate 통과 |
| 컨테이너화 | Docker Compose로 단일 명령 실행 |
| CI/CD | Jenkins 파이프라인 자동 빌드·배포 |

## 6. MVP 인수 기준

| # | 항목 | 기준 |
|---|------|------|
| AC-01 | 인증 | 미로그인 사용자는 Todo 기능 접근 불가 |
| AC-02 | 회원 가입 | 중복 이메일 가입 시 400 오류 반환 |
| AC-03 | Todo 추가 | 제목 없이 추가 시 422 검증 오류 반환 |
| AC-04 | 기한 초과 표시 | 완료일 < 오늘인 미완료 항목은 빨간 강조 |
| AC-05 | 데이터 격리 | 사용자 A의 Todo는 사용자 B에게 보이지 않음 |
| AC-06 | 검색 ✅ | 키워드 입력 시 제목에 포함된 항목만 표시, 결과 개수 안내 |
| AC-07 | 인라인 수정 ✅ | 제목 클릭 → 편집 → Enter 저장 / ESC 취소 |
| AC-08 | 통계 | 완료율이 실시간으로 업데이트 (미구현) |
