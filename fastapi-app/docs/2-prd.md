# PRD (Product Requirements Document)

## My TodoList 앱

---

## 1. 제품 비전 및 목표

### 비전

"누구나 쉽게 할 일을 관리하고, 기한을 놓치지 않는 개인 생산성 도구"

### 목표

| 목표                                         | 측정 기준                        |
| -------------------------------------------- | -------------------------------- |
| 학생 사용자의 과제/일정 관리 효율 향상       | 할 일 완료율 70% 이상            |
| 직관적인 UI로 5분 내 온보딩 완료             | 첫 Todo 등록까지 소요 시간 ≤ 5분 |
| 기한 초과 항목 시각적 구분으로 미처리율 감소 | 기한 초과 인지율 100%            |

## 3. 기능 범위 (In Scope / Out of Scope)

### In Scope (MVP 포함)

- 회원 가입 (이메일 + 비밀번호)
- 로그인 / 로그아웃 (JWT 인증)
- 할 일 추가 (제목, 설명, 계획 완료일, 우선순위)
- 할 일 목록 조회 (본인 것만)
- 할 일 수정
- 할 일 삭제
- 완료 상태 토글
- 계획 완료일 경과 항목 시각적 강조 (빨간 테두리/배경)

### Out of Scope (향후 고려)

- 소셜 로그인 (Google, Kakao)
- 팀/공유 할 일
- 알림/푸시 메시지
- 모바일 네이티브 앱
- 파일 첨부
- 반복 일정 (매주, 매일 등)
- 통계 대시보드

## 5. UI/UX 요구사항 및 화면 흐름

### 화면 목록

1. **로그인 페이지** (`/login`)
2. **회원 가입 페이지** (`/register`)
3. **메인 페이지 — 할 일 목록** (`/`)

### 화면 흐름

```
[비인증 접근] → 로그인 페이지
     │
     ├─ 로그인 성공 → 메인 페이지 (할 일 목록)
     │                    │
     │                    ├─ 할 일 추가 (모달/인라인 폼)
     │                    ├─ 완료 토글 (체크박스 클릭)
     │                    ├─ 수정 (편집 아이콘 → 인라인 수정)
     │                    └─ 삭제 (휴지통 아이콘 → 확인 다이얼로그)
     │
     └─ 회원 가입 링크 → 회원 가입 페이지 → 가입 완료 → 로그인 페이지
```

### UI 공통 규칙

- 기한 초과 항목: 빨간 왼쪽 테두리 + 연한 빨간 배경
- 완료된 항목: 텍스트 취소선 + 회색 처리
- 우선순위 표시: `High` 빨강 배지, `Medium` 노랑 배지, `Low` 초록 배지
- 반응형: 모바일 320px 이상 지원

## 7. MVP 인수 기준

| #     | 항목           | 기준                                         |
| ----- | -------------- | -------------------------------------------- |
| AC-01 | 인증           | 미로그인 사용자는 Todo 기능 접근 불가        |
| AC-02 | 회원 가입      | 중복 이메일 가입 시 409 오류 반환            |
| AC-03 | Todo 추가      | 제목 없이 추가 시 422 검증 오류 반환         |
| AC-04 | 기한 초과 표시 | 완료일 < 오늘인 미완료 항목은 빨간 강조 표시 |
| AC-05 | 데이터 격리    | 사용자 A의 Todo는 사용자 B에게 보이지 않음   |
| AC-06 | 완료 토글      | 완료 처리 후 취소선 즉시 반영                |
| AC-07 | 삭제           | 삭제 후 목록에서 즉시 제거                   |
