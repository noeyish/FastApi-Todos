# 도메인 정의서 (Domain Definition)
## My TodoList 앱 — v5.0.0

---

## 핵심 기능

- 회원등록은 누구나 가능하다
- 등록된 회원은 로그인 할 수 있다
- 인증된 사용자만 할 일을 추가/조회/수정/삭제할 수 있다
- 할 일 등록 시 계획완료일, 우선순위를 함께 등록할 수 있다
- 계획완료일을 지난 할 일은 시각적으로 구분되어 표시된다
- 할 일 목록은 필터(전체/진행 중/완료)와 정렬(우선순위/날짜)을 지원한다
- 할 일 제목으로 검색할 수 있다 _(v5 신규)_
- 완료율 및 우선순위별 통계를 대시보드에서 확인할 수 있다 _(v5 신규)_
- 모든 사용자는 인증(로그인) 후 애플리케이션 기능을 이용한다

---

## 핵심 도메인 개념

| 도메인 객체 | 설명 | 주요 속성 |
|------------|------|----------|
| **User** | 앱을 사용하는 회원 | id, email, hashed_password |
| **Todo** | 사용자의 할 일 항목 | id, title, description, priority, due_date, completed, user_id |

---

## 비즈니스 규칙

| 규칙 ID | 내용 |
|---------|------|
| BR-01 | 미인증 사용자는 Todo 기능에 접근할 수 없다 |
| BR-02 | 이메일은 전체 시스템에서 유일해야 한다 |
| BR-03 | 사용자는 본인의 Todo만 조회/수정/삭제할 수 있다 |
| BR-04 | 제목은 Todo의 필수 항목이다 |
| BR-05 | 우선순위는 low / medium / high 중 하나여야 한다 |
| BR-06 | 완료일이 오늘보다 이전인 미완료 항목은 기한 초과로 표시한다 |

## 평가 기준

| 항목 | 기준 |
|------|------|
| 완전성 | 모든 핵심 영역(인증, CRUD, 검색, 통계) 포함 |
| 명확성 | 모호함 없는 비즈니스 규칙 정의 |
| 추적성 | PRD 사용자 스토리와 1:1 연결 가능 |
| 검증 가능성 | 각 기능은 테스트 케이스로 검증 가능 |
| 유지보수성 | 기능 추가 시 해당 섹션만 업데이트 |
